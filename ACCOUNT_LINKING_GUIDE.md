# 账号合并和邮箱绑定完整指南

## 问题背景

当您在不同的时间使用不同的登录方式：
- **第一次：** 用邮箱 (email@example.com) + 密码登录 → 创建了账户 A
- **第二次：** 用 Google 账号登录 → 创建了账户 B  
- **现在：** 在账户 B 上想绑定邮箱 A，但得到错误：

```
❌ Please verify the new email before changing email.
❌ email-already-in-use
```

**原因：** Firebase 中一个邮箱只能关联一个账户。您的邮箱已经被账户 A 使用了。

---

## 解决方案

我们现在提供了**自动账号合并**功能，可以将多个登录方式链接到同一个账户。

### 工作原理

```
情况：
- 账户 A：email@example.com + 密码
- 账户 B：Google 登录（当前）

解决：
账户 B 链接邮箱凭证
     ↓
识别邮箱已被使用
     ↓  
用户输入邮箱密码
     ↓
验证密码
     ↓
链接成功：账户 B 现在可以用邮箱或 Google 登录
```

---

## 使用步骤

### 步骤 1：打开联系方式管理

1. 登录到应用（用任何方式）
2. 点击左上方 ⋮ 菜单
3. 选择 "联系方式管理"

### 步骤 2：输入要绑定的邮箱

1. 在 "邮箱" 部分输入邮箱地址
2. 点击 "绑定邮箱"

### 步骤 3：系统会检查邮箱

有两种结果：

#### ✅ 情况 A：邮箱未被使用

```
✓ 邮箱绑定成功！
✓ 验证邮件已发送
→ 直接完成，无需额外步骤
```

#### ⚠️ 情况 B：邮箱已被使用

```
⚠️ 该邮箱已被使用
⚠️ 需要进行账号链接

系统会显示对话框，提供两个选项：
1. 直接绑定（推荐）
2. 通过密码链接
```

### 步骤 4A：直接绑定（推荐）

**如果您仍有原账户的登录方式：**

1. 点击 "取消"
2. 切换到该邮箱关联的登录方式
   - 如果是邮箱+密码：用邮箱和密码重新登录
   - 如果是 Google：用原来的 Google 账号登录
3. 系统会自动检测这是同一个邮箱，**自动合并账户**

### 步骤 4B：通过密码链接

**如果您知道该邮箱的密码：**

1. 点击对话框中的 "使用密码链接"
2. 系统显示密码输入框
3. 输入该邮箱关联账户的密码
4. 点击 "链接账户"
5. 系统会验证密码，链接成功

**完成后：** 您的当前账户现在可以用两种方式登录：
- 原始方式（Google / 手机号等）
- 邮箱 + 密码

## 具体场景演示

### 场景 1：Google 账户想绑定邮箱

```
当前账户创建方式：Google Sign-In
想绑定的邮箱：user@example.com
邮箱状态：曾经用 user@example.com + 密码登录过

操作步骤：
1. 打开联系方式管理
2. 在邮箱部分输入 user@example.com
3. 点击 "绑定邮箱"
4. 系统提示 "邮箱已被使用"
5. 选择 "使用密码链接"
6. 输入 user@example.com 的密码
7. 点击 "链接账户"
8. ✅ 账户链接成功！

之后：
- 可以用 Google 登录
- 也可以用 user@example.com + 密码登录
- 登录后是同一个账户（数据、记录都相同）
```

### 场景 2：手机号账户想绑定邮箱

```
当前账户创建方式：手机号 + 验证码
想绑定的邮箱：myemail@gmail.com  
邮箱状态：新邮箱，未被使用过

操作步骤：
1. 打开联系方式管理
2. 在邮箱部分输入 myemail@gmail.com

5. 系统发送验证邮件
6. 点击邮件中的验证链接

就这样，完成了！
```

### 场景 3：已验证邮箱的更新

```
当前账户：已有邮箱 old@example.com（已验证）
想更换为：new@example.com

建议步骤：
1. 解绑当前邮箱 → 点击 "解绑"
2. 绑定新邮箱：
   - 如果 new@example.com 未被使用：直接绑定
   - 如果 new@example.com 已被使用：按照场景 1 的步骤链接
```

## 常见问题

### Q1: 链接失败，显示 "密码错误"

**A:** 
- 确认密码确实是该邮箱账户的密码
- 注意大小写
- 如果忘记密码，可以：
  1. 取消此操作
  2. 用该邮箱登录 → 重置密码
  3. 重新回来绑定

### Q2: 链接失败，显示 "该邮箱未注册"

**A:**
- 您输入的邮箱从未被使用过
- 检查是否拼写错误
- 如果是新邮箱，直接点击 "绑定邮箱" 即可

### Q3: 链接失败，显示 "该邮箱已与其他账户关联"

**A:**
- 该邮箱可能被另一个 Google 或其他登录方式使用了
- 如果是您自己的邮箱，使用关联的登录方式登录，系统会自动识别并合并

### Q4: 我有两个账户，想合并成一个

**A:**
该功能现在已经完全支持！有三种方式：

**方式 1：单账户链接多个登录方式**
```
账户 A：Google 登录
账户 B：邮箱 + 密码

要合并成一个账户？
1. 用账户 A 登录
2. 在联系方式管理中，绑定账户 B 的邮箱
3. 输入账户 B 的密码链接
4. 完成！账户 A 现在有两种登录方式
```

**方式 2：共享数据的两个账户**
```
当前暂不支持合并数据
每个账户的记录是独立的
```

### Q5: 链接后数据会发生什么？

**A:**
链接只是添加登录方式，不合并数据：

```
账户 A（邮箱登录）
  ├── 记录 1：家庭相册
  ├── 记录 2：重要文件
  └── ...

账户 B（Google 登录）  
  ├── 记录 A：工作文档
  ├── 记录 B：旅游照片
  └── ...

链接 A 的邮箱到 B 后：
账户 B 现在可以用两种方式登录：
  ├── Google
  └── A 的邮箱 + 密码

登录时看到的是账户 B 的记录（记录 A、B...）
```

**不会自动合并数据。**

## 最佳实践

### ✅ 推荐做法

1. **尽早链接可用的登录方式**
   ```
   刚创建账户时，就链接邮箱、手机号等
   万一其中一个登陆方式不可用，还有备选
   ```

2. **定期验证邮箱**
   ```
   绑定邮箱后，立即点击验证链接
   这样邮箱就不会"未验证"显示
   ```

3. **保管好密码**
   ```
   如果想以后用密码链接，要记住密码
   或定期重置密码确保安全
   ```

4. **多种登录方式**
   ```
   不要只依赖一种登录方式
   尽量绑定邮箱 + 手机号
   这样最安全
   ```

### ❌ 避免做法

1. **不要在多个账户使用同一个邮箱**
   ```
   这会导致冲突和混淆
   ```

2. **不要忘记验证邮件**
   ```
   邮箱绑定后要及时验证
   否则在不同设备上可能有问题
   ```

3. **不要随意解绑**
   ```
   如果只有一种登录方式，不要解绑
   万一登陆方式失效就无法登录了
   ```

## 安全说明

### 邮箱安全
- ✅ 邮箱验证是必需的
- ✅ 验证链接有时间限制（24 小时）
- ✅ 只有验证后的邮箱才算真正绑定

### 密码安全  
- ✅ 密码只用于验证，不会被保存
- ✅ 密码尝试次数有限制（防止暴力破解）
- ✅ 验证成功后会链接邮箱凭证
- ✅ 之后可以直接用邮箱 + 密码登录（无需验证码）

### 账户安全
- ✅ 链接不会合并数据，只是添加登录方式
- ✅ 每个登录方式都能访问同一个账户
- ✅ 可以随时解绑某个登录方式（保留至少一个）

## 技术细节

### 实现方式

```dart
// 检查邮箱是否已被使用
final status = await authService.checkEmailStatus(email);

if (status['available'] == true) {
  // 邮箱未被使用，直接绑定
  await authService.bindEmail(email);
} else {
  // 邮箱已被使用，需要链接
  await authService.linkEmailCredential(email, password);
}
```

### 验证流程

```
1. 用户输入邮箱
   ↓
2. 检查邮箱状态
   ├─ 未使用 → 直接绑定 → 发送验证邮件 ✅
   └─ 已使用 → 显示链接选项
                ├─ 输入密码 → 验证 → 链接 ✅
                └─ 取消 → 返回
```

```
├── phoneNumberVerified: true/false
├── linkedEmails: ["email1@example.com", "email2@example.com"]
└── linkedPhones: ["+8613800138000", "+8613800138001"]
```

## 常见错误代码解释

| 错误 | 原因 | 解决方案 |
|------|------|--------|
| `email-already-in-use` | 邮箱被其他账户使用 | 用关联的登录方式登录，或输入密码链接 |
| `wrong-password` | 密码错误 | 检查密码是否正确，注意大小写 |
| `user-not-found` | 邮箱未注册 | 邮箱可能拼写错误或从未注册过 |
| `requires-recent-login` | 需要重新验证 | 重新登录后重试 |
| `too-many-requests` | 尝试次数过多 | 请稍后再试（通常 1 小时后） |
