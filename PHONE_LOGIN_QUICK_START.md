# 手机号登录功能 - 快速开始

## 🚀 快速开始（5分钟）

### 第一步：Firebase 配置（一次性）

1. **打开 Firebase Console**
   - 访问 https://console.firebase.google.com
   - 选择您的项目

2. **启用 Phone Authentication**
   - 左侧菜单 → Authentication
   - 点击 "Sign-in method" 标签
   - 找到 "Phone" → 点击启用 ✅

3. **就这样！** 无需其他配置即可开始使用

### 第二步：在应用中使用

#### 👤 手机号登录
1. 打开应用 → 点击 "使用手机号登录" 按钮
2. 输入手机号（如 13800138000）
3. 点击 "发送验证码"
4. 等待短信中的验证码
5. 输入验证码 → 点击 "登录"

#### 📧 绑定邮箱和手机号
1. 登录后 → 右上角 ⋮ 菜单
2. 选择 "联系方式管理"
3. **绑定邮箱：**
   - 输入邮箱地址
   - 点击 "绑定邮箱"
   - 查收验证邮件
4. **绑定手机号：**
   - 输入手机号
   - 点击 "发送验证码"
   - 输入验证码
   - 完成绑定

## 📱 支持的手机号格式

- `13800138000` ✅（国内号码，自动识别为中国）
- `+8613800138000` ✅（完整格式）
- `+1234567890` ✅（其他国家格式）

## 🗄️ 用户数据存储

用户的联系方式信息存储在 Firestore 中：

```
Firestore:
└── users/
    └── {uid}/
        ├── email: "user@example.com"
        ├── emailVerified: true/false
        ├── phoneNumber: "+8613800138000"
        ├── phoneNumberVerified: true/false
        ├── createdAt: 2024-01-01T12:00:00Z
        └── lastLoginAt: 2024-01-15T08:30:00Z
```

## ⚙️ 常见设置

### 修改验证码有效期（可选）
在 `auth_service.dart` 中修改：
```dart
timeout: const Duration(minutes: 2), // 改为其他值
```

### 修改验证码长度（可选）
在 `phone_login_page.dart` 和 `contact_binding_page.dart` 中修改：
```dart
maxLength: 6, // 改为其他值
```

### 修改默认国家代码（可选）
在 `phone_login_page.dart` 中修改：
```dart
if (!phoneNumber.startsWith('+')) {
  phoneNumber = '+886$phoneNumber'; // 改为台湾或其他国家
}
```

## 🐛 调试技巧

### 查看登录日志
- 打开 Flutter 开发工具的控制台
- 搜索 🔐 或 ❌ 来快速定位日志

### 查看 Firebase 日志
- 打开 Firebase Console
- Authentication → Logs
- 查看最近的登录尝试记录

### 常见错误
| 错误 | 解决方案 |
|------|----------|
| "验证码无效" | 确保输入的是完整的 6 位验证码 |
| "验证码已过期" | 点击"重新发送"获取新验证码 |
| "发送失败" | 检查手机号格式和网络连接 |
| "用户未登录" | 确保先成功登录 |

## 🔐 安全建议

1. **生产环境启用 reCAPTCHA**
   - Firebase Console → Authentication → reCAPTCHA Enterprise
   - 需要额外配置和费用

2. **启用 SMS 限制**
   - Firebase Console → Phone → Quota enforcement
   - 防止滥用验证码

3. **定期审查登录日志**
   - 监控异常登录活动

## 💡 进阶用法

### 自定义验证错误处理
在 `auth_service.dart` 中的 `verificationFailed` 回调中处理不同类型的错误。

### 与现有认证系统集成
所有登录后，用户信息会自动保存到 Firestore 的 users 集合中。

### 实现双因素认证
可以在绑定手机号后，使用 `phoneNumber` 作为第二认证因素。
