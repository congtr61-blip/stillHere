## 🔧 构建 APK/AAB 的命令

### 构建发布版本

```bash
# 第 1 步：清理项目
flutter clean
flutter pub get

# 第 2 步：构建 AAB（推荐，Google Play 标准）
flutter build appbundle --release

# 或构建 APK（用于测试）
flutter build apk --release --split-per-abi

# 输出位置：
# AAB: build/app/outputs/bundle/release/app-release.aab
# APK: build/app/outputs/flutter-apk/app-release.apk
```

### 查看文件大小

```bash
# Windows PowerShell
Get-ChildItem "build/app/outputs/bundle/release/" | Select-Object Name, Length
```

## 🆘 常见问题

### Q1：发布到 Google Play 需要多少钱？
**A：** 
- 开发者账户：一次性 $25 USD
- 应用发布：免费
- 应用内购买：Google 收取 30% 佣金

### Q2：Apple App Store 怎么发布？
**A：** 
- 需要 Mac 电脑（不能在 Windows 构建 iOS）
- 需要 Apple 开发者账户（$99/年）
- 需要 iOS 签名证书

### Q3：隐私政策在哪里发布？
**A：** 
- 方案 1：GitHub Pages（免费）
- 方案 2：单独的网站（如 Wix、Webflow）
- 方案 3：在你的网站上
- 然后在 Google Play Console 中提供 URL

### Q4：可以在 Windows 上构建安卓应用吗？
**A：** 是的！
```bash
flutter build apk --release     # 构建 APK
flutter build appbundle --release  # 构建 AAB
```

### Q5：可以先发布测试版本吗？
**A：** 可以！Google Play 支持多种发布方式：
- **内部测试**（最多 100 人）
- **闭路测试**（最多 10,000 人）
- **开放测试**（不限制人数）
- **生产发布**（所有用户）
